# بورتفوليو عبدالرحمن الربيعي

موقع بورتفوليو شخصي عربي، سريع ومتجاوب، ومصمم للنشر المباشر عبر **GitHub Pages**. لا يحتاج إلى تثبيت حزم أو خطوة بناء؛ افتح `index.html` في المتصفح أو ارفعه إلى GitHub كما هو.

> هذا المستودع يعرض بورتفوليو عبدالرحمن الربيعي — طالب هندسة الحاسب المهتم بتطوير الويب، الذكاء الاصطناعي، والرؤية الحاسوبية. يحتوي الموقع على سيرة مختصرة، قائمة مهارات، ومشروع مُنفّذ فعليًا: DriveThruGestureAI.

## المميزات

- عرض مزدوج اللغة (عربي/إنجليزي) مع تبديل يحفظ تفضيل المستخدم.
- صفحة سريعة وخفيفة بدون اعتماد على أدوات بناء أو مكتبات خارجية.
- تصميم متجاوب يدعم الأجهزة الصغيرة والكبيرة مع وضع داكن/فاتح محفوظ محليًا.
- مشروع مميز: DriveThruGestureAI — نظام طلبات بدون لمس باستخدام الذكاء الاصطناعي والرؤية الحاسوبية.
- بنية HTML دلالية مع اعتبارات وصولية أساسية.

## حول المحتوى

الموقع مُعد لعرض معلومات شخصية ومشاريع حقيقية. القالب يحافظ على التصميم والأداء بينما تعرض الصفحات بياناتك ومشروعك الفعلي DriveThruGestureAI. البريد وروابط GitHub مضبوطة على حساب صاحب المشروع.

لا تشارك معلومات حساسة مثل كلمات المرور أو مفاتيح API في المستودع العام.

## تشغيل محلي

يكفي فتح `index.html` بالنقر المزدوج أو استخدام Live Server في VS Code لعرض التغييرات مباشرة.

## النشر على GitHub Pages

1. أنشئ مستودعًا جديدًا في GitHub، مثل `portfolio`، واجعله **Public** إذا كان حسابك المجاني لا يوفّر Pages للمستودعات الخاصة.
2. ارفع جميع هذه الملفات إلى جذر المستودع، وتأكد من وجود `index.html` و`.nojekyll` في الجذر.
3. من المستودع افتح **Settings** ثم **Pages**.
4. في **Build and deployment** اختر **Deploy from a branch**.
5. اختر فرع `main` ومجلد `/(root)` ثم اضغط **Save**.
6. انتظر دقيقة أو دقيقتين. سيظهر رابط الموقع في صفحة Pages، غالبًا بالشكل:

   ```text
   https://YOUR-GITHUB-USERNAME.github.io/portfolio/
   ```

7. افتح الرابط من الجوال ومن نافذة خاصة للتأكد من أن التحديث منشور للزوار.

### النشر باستخدام Git (اختياري)

نفّذ الأوامر التالية داخل مجلد المشروع بعد إنشاء المستودع الفارغ، مع استبدال الرابط برابطك:

```powershell
git init
git add .
git commit -m "Create personal portfolio"
git branch -M main
git remote add origin https://github.com/YOUR-GITHUB-USERNAME/portfolio.git
git push -u origin main
```

## مراجعة المخاطر قبل النشر

| الخطر أو المشكلة | الإجراء المطبق | ما عليك فعله |
| --- | --- | --- |
| ظهور بيانات حساسة في مستودع عام | لا توجد أسرار أو تكاملات خارجية في المشروع، و`.gitignore` يستثني ملفات البيئة | راجع النصوص والصور قبل الرفع |
| تعطل النموذج بسبب عدم وجود خادم | استخدمت رابط بريد مباشر بدل نموذج يوحي بإرسال ناجح | ضع بريدك الحقيقي أو أضف خدمة نماذج موثوقة لاحقًا |
| بطء الموقع أو تعطل الخطوط | لا يوجد إطار عمل أو صور خارجية أو اعتماد على CDN | اضغط أي صور تضيفها واستخدم WebP/AVIF |
| عدم ملاءمة الجوال | التصميم متجاوب واُختبر على قياسات صغيرة | افحص موقعك بعد وضع صور المشاريع الفعلية |
| روابط غير صحيحة | الروابط الحالية علامات واضحة يجب تغييرها | استبدل جميع `your-...-username` قبل الإطلاق |
| نشر صفحة قديمة | GitHub Pages قد يحتاج قليلًا من الوقت أو يحتفظ بذاكرة مؤقتة | تحقق من أحدث commit ومن إعداد `main / root` |
| مسار أو رابط غير موجود | أضفت صفحة `404.html` وروابط نسبية مناسبة لمجلد المشروع | اختبر الروابط بعد النشر، خصوصًا إذا غيّرت اسم المستودع |
| فهرسة أو مشاركة غير مقصودة | الموقع عام عند نشره في مستودع عام | لا تنشر ما لا تريد أن يراه أي شخص |

## قائمة فحص أخيرة

- [ ] الاسم، البريد، وروابط التواصل صحيحة.
- [ ] كل مشروع يعرض أعمالك ونتائجك أنت، وليس النص النموذجي.
- [ ] لا توجد معلومات حساسة أو صور لا تملك حق نشرها.
- [ ] جربت أزرار القائمة والمشاريع والوضع الفاتح والداكن.
- [ ] راجعت الموقع على الجوال.
- [ ] اخترت `main` و`/(root)` في إعدادات GitHub Pages.
